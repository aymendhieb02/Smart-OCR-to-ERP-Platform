# Release v1.1 - Hybrid Advisory

Release date: 2026-07-21

## Summary

This release freezes the Smart OCR-to-ERP Platform as a deterministic invoice Document AI pipeline with an optional, evidence-grounded local LLM advisory layer.

The deterministic engine remains the source of truth. The LLM is a bounded review assistant used only for ambiguous low-confidence cases, with the current deployment policy limited to supplier/customer recovery when those fields are missing.

## Deterministic Baseline

Current deterministic benchmark summary:

- Invoice number: 100%
- Invoice date: 100%
- Supplier canonical: 70%
- Customer canonical: 100%
- Amount TTC: 76.92%
- Line-item presence: 60%
- Exact row count: 52%
- Within +/-1 rows: 68%

These numbers describe deterministic extraction behavior. They do not prove hybrid accuracy improvement.

## Hybrid Architecture

The hybrid advisory layer adds:

- structured evidence package generation;
- prompt variants for local Ollama models;
- JSON response parsing;
- safe correction proposal validation;
- correction gate enforcement;
- review assistant explanations;
- benchmark/report tooling for prompt calibration, ground-truth quality, accuracy gating, error taxonomy, ROI, and deployment recommendation.

The LLM receives structured JSON evidence. It does not receive raw uncontrolled OCR text and cannot bypass deterministic validation.

## Safety Controls

- LLM disabled by default.
- Advisory mode recommended.
- Auto-apply disabled.
- Evidence references required.
- Unsupported corrections rejected.
- Protected high-confidence deterministic fields protected from silent overwrite.
- Deterministic validation runs after corrections.
- ERP readiness remains deterministic.
- Human reviewer remains in control.

## Known Limitations

- Verified ground truth is incomplete.
- Hybrid accuracy improvement is therefore not proven yet.
- Current hybrid conclusions are provisional and based on gate behavior plus incomplete labels.
- Local LLM inference is slow on CPU-only machines.
- Stage-specific LLM latency is not fully instrumented in saved benchmark artifacts.
- Table and financial corrections should not be auto-applied.

## Deployment Recommendation

Deploy only in advisory mode.

Current routing policy:

- call the LLM only when supplier or customer is missing;
- keep OCR, invoice number, dates, totals, VAT, tables, validation, and ERP readiness deterministic;
- do not enable global auto-apply;
- do not enable table auto-correction;
- do not enable financial/totals auto-correction.
